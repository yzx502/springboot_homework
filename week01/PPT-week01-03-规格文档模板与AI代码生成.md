## 为什么要写规格文档？（1）

从“人”的角度：

- 团队协作需要**共同语言**：  
  - 前后端、测试、产品需要看到同一份需求说明；  
  - 代码只是实现，规格文档才是“约定”。  

从“AI”的角度：

- 大模型需要**明确的指令**：  
  - 模糊描述 → 模糊代码；  
  - 清晰文档 → 比较稳定、可控的代码输出。  

---

## 为什么要写规格文档？（2）

不写文档或文档不清的常见后果：

- 同一个接口在不同人眼中含义不同；  
- 测试不知道如何设计用例；  
- AI 生成的代码“看起来能跑”，但实际业务逻辑对不上；  
- 后续维护时没人记得当初是怎么约定的。  

本课程的基本原则：

> **先写文档，再写代码；  
> 文档描述不了的需求，就不要急着写代码。**

## 初级规格文档模板结构（总览）

五大核心章节：

1. **核心目标**  
   - 用 1–2 句话说明这个接口/模块“要解决什么问题”。  
2. **业务规则**  
   - 功能要遵守的业务逻辑、边界条件、异常场景。  
3. **技术约束**  
   - 使用的框架版本、技术栈、限制条件等。  
4. **输入输出**  
   - 清晰列出请求参数和返回结果（含示例）。  
5. **验收标准**  
   - 如何判断“做完了”“合格了”。

## 模板展开示例：核心目标

模板示例：

```markdown
## 1. 核心目标
- 用 1 句话明确功能价值
```

Hello 接口示例：

```markdown
## 1. 核心目标
- 开发一个 Hello 接口，返回统一格式的欢迎信息，用于验证 Spring Boot 项目的基本配置是否正确。
```

说明：

- 不求写得很长，但要**具体**，避免只写“实现一个接口”。

## 模板展开示例：业务规则

模板示例：

```markdown
## 2. 业务规则
- 功能要实现的具体业务规则
- 边界条件和异常场景
```

Hello 接口示例：

```markdown
## 2. 业务规则
- 接口路径：GET /api/hello
- 无需请求参数
- 返回 JSON，包含 code、msg、data 三个字段
- code 固定为 200，表示成功
- msg 固定为 "success"
- data 为欢迎语字符串，例如："Hello Spring Boot"
```

## 基础代码生成 Prompt 模板

在课堂中展示基础 Prompt 示例（文字版）：

```text
基于以下规格文档，生成 Spring Boot 3.x 的 Controller 和 ResultVO 代码：

【规格文档开始】
（在这里粘贴 hello-spec.md 的全文）
【规格文档结束】

生成要求：
1. 创建一个 HelloController，使用 @RestController 注解，类上使用 @RequestMapping("/api")。
2. 提供 GET /hello 接口方法，方法名为 hello。
3. 返回类型为 ResultVO<String>。
4. 若项目中尚未定义 ResultVO 类，请一并生成 ResultVO 通用返回类代码。
5. 不需要解释，只输出 Java 代码。
```

强调：

- 文档 **越清晰**，生成的代码越接近预期。

## “文档→AI→代码”的操作流程

用流程图或步骤列表展示：

1. 在本地编写并保存 `hello-spec.md`；  
2. 打开大模型网页；  
3. 复制 Prompt 模板 + 规格文档内容；  
4. 粘贴到对话框并发送；  
5. 复制返回的 `HelloController` 和 `ResultVO` 代码；  
6. 粘贴到 `springboot-hello` 项目对应的包中；  
7. 检查是否有编译错误。  

---

## 文档写得不好会怎样？（反例）

展示一个“糊涂文档”的例子（简化版）：

```markdown
## 2. 业务规则
- 做一个接口，返回 hello
```

可能导致的问题：

- AI 可能不知道路径、方法、参数、返回格式；  
- 得到的代码风格、命名、返回结构都不统一；  
- 以后维护时难以对齐到统一标准。  

结论：

> 花在写文档上的时间，会在后面测试与返工中“加倍赚回来”。

## 课堂实操说明

本节课后半段安排：

- 每位同学：  
  - 在 `week01/docs/` 下编写自己的 `hello-spec.md`；  
  - 使用大模型按 Prompt 生成 `HelloController` 与 `ResultVO`；  
  - 粘贴到 `springboot-hello` 工程中，确保无编译错误。  

注意事项：

- 编写文档时先专注内容，不必追求一次完美；  
- 如果 AI 生成代码与你预期不符，优先检查**文档是否写清楚了**。
