# 实验任务单：课程导入与工具准备（week01-课时1）

## 一、实验目标

1. 明确本课程的学习目标、考核方式和综合实训「江苏中医知识服务云平台」项目要求。  
2. 完成个人开发环境与 AI 工具的基础准备自查。  
3. 形成对“文档驱动 + AI 协同”学习模式的初步理解。  

## 二、实验环境

- 个人电脑（已预装 JDK / Git / IDE）  
- 浏览器
- 课程 Git 仓库地址

## 三、实验任务

### 任务 1：填写个人学习预期与自评

1. 在个人仓库 `week01` 目录下新建文件：`learning_expectation.md`。  
2. 按以下提示撰写（不少于 300 字）：
   - 你目前的 Java 和数据库基础情况（学过哪些内容，熟练程度如何）；  
   - 你对本课程的期待（希望掌握什么能力、完成什么样的项目）；  
   - 你更倾向于选择的任务难度层级（基础/挑战）及原因；  
   - 你在「江苏中医知识服务云平台」中最想实现的 1–2 个功能。  
3. 写完后保存，并提交到个人 Git 仓库。  

> 建议：用列表或小节标题清晰组织内容，后续会据此进行分层与分组参考。

### 任务 2：开发环境初步检查

1. 打开命令行/终端，依次执行：

```bash
java -version
git --version
```

2. 观察输出结果，在 `learning_expectation.md` 末尾增加一节“环境检查”，记录：
   - Java 实际版本号；  
   - Git 实际版本号；  
   - 若命令无法识别，简要描述错误信息（便于教师后续帮助排查）。  

3. 打开 IntelliJ IDEA，确认能进入欢迎界面（无需创建项目）。  

### 任务 3：AI 工具连通性检查

1. 尝试访问至少一个指定的大模型网站。  

2. 若成功登录或可使用网页版 Chat：
   - 在对话框中输入一句话：  
     > “请帮我用 3 句话介绍什么是 Spring Boot。”  
   - 将返回结果复制，粘贴到 `learning_expectation.md` 最后，标题为“AI 回答示例”。  

## 四、提交要求

本课结束前或课后指定时间内，完成以下提交：

- 文件路径与内容：
  - `week01/learning_expectation.md`：包含  
    - 个人基础与期待；  
    - 任务难度倾向；  
    - 江苏中医知识服务云平台感兴趣的功能；  
    - 环境检查结果；  
    - AI 回答示例。  
- 截图：  
  - `week01/screenshots/env-check.png`：命令行 `java -version` 与 `git --version` 的截图。  

## 五、评价方式

- **合格标准：**
  - 文档存在且字数 ≥ 300；  
  - 明确填写了基础情况与学习期待；  
  - 环境检查信息完整（无论成功或失败）。  
- **加分点（用于平时表现）：**
  - 对江苏中医知识服务云平台功能有较深入或有创意的构思；  
  - 对“文档驱动 + AI 协同”模式有自己的理解或疑问。  

## 六、自动化检查建议（供教师配置 CI 使用）

在课程仓库 CI 脚本中，可对学生仓库进行简单自动检查：

1. 检查文件是否存在：

```bash
test -f week01/learning_expectation.md
```

2. 检查字数（示例，按行数或字符数粗略判断）：

```bash
wc -w week01/learning_expectation.md
```

> 若字数低于设定阈值（如 300 字），在 CI 报告中给出提示“学习预期字数偏少”。  

3. 后续可扩展：
   - 解析文档中是否包含“环境检查”、“AI 回答示例”等标题，用于半自动检查完成度。
