## 本课目标与流程

本课目标：

- 每位同学至少完成**一档任务**：基础/挑战；  
- 初步体验完整的“文档 → AI → 代码 → 运行验证”流程；  
- 按规范提交“文档 + 代码 + 截图”。  

本课流程：

1. 说明任务与提交要求（约 5 分钟）；  
2. 学生自主选择任务难度并开始实践（约 25 分钟）；  
3. 成果展示与课堂点评（约 10 分钟）；  
4. 布置课后整理与提交任务（约 5 分钟）。

## 任务总览

表格展示：

| 难度 | 任务名称 | 接口示例 | 核心内容 |
| ---- | -------- | -------- | -------- |
| 📚 基础版 | Hello 接口巩固 | `GET /api/hello` | 完整文档+代码+运行 |
| 🚀 挑战版 | 健康检查 + 系统信息 + 环境适配 | `/api/health`、`/api/system/*` | Health 必做 + 多接口 + 多环境配置 + 异常处理 |

说明：

- 两档任务共享同一项目（`springboot-hello`），只是在功能复杂度和文档要求上逐步增加。  

---

## 基础版任务：Hello 接口

目标：

- 确保每位同学都能完成一个“文档驱动 + AI 生成”的完整示例。  

要求回顾：

- 规格文档：`week01/docs/hello-spec.md` 完整、清晰；  
- 代码：`HelloController` + `ResultVO` 正常编译；  
- 运行：  
  - 启动项目；  
  - 访问 `GET /api/hello` 返回统一格式 JSON；  
  - 截图保存。  

建议对象：

- 对 Java 或 Spring Boot 还不太熟悉的同学；  
- 希望先把流程跑通、打好基础的同学。  

---

## 挑战版任务：健康检查 + 系统信息 + 环境适配

目标：

- 在基础版基础上，开发系统健康检查接口及扩展接口。  

必做：`GET /api/health`，返回 appName、version、time、status。  

选做（至少完成 1 项）：  

- `GET /api/system/info`：返回应用信息；  
- `GET /api/system/env`：返回部分**安全的**环境变量；  
- `GET /api/system/metrics`：返回 JVM 运行指标。  

扩展要求：  

- 多环境配置（dev/test/prod）；  
- 简单全局异常处理。  

适合对象：

- 已熟悉基础版内容；  
- 想要挑战更复杂需求与配置的同学。

## 实战时间与分层辅导安排

建议课堂时间分配（45 分钟）：  

- 0–5 分钟：说明任务与提交规范；  
- 5–30 分钟：学生自主实践，教师巡回指导：  
  - 对基础版学生：  
    - 重点解决环境与运行问题；  
    - 确保能跑通 Hello 接口。  
  - 对挑战版学生：  
    - 讨论多环境配置与安全性问题；  
    - 提点 JVM 指标获取方式等。  
- 30–40 分钟：成果展示（每个难度层级至少 1 人展示）；  
- 40–45 分钟：教师点评与课后任务布置。  

---

## 成果展示与点评标准

展示内容建议：

- 简要说明自己选择的任务难度和实现思路；  
- 展示规格文档中的关键部分（业务规则、输入输出）；  
- 在浏览器/Apifox 中演示接口调用结果。  

点评重点：

- 文档是否清晰、可执行；  
- 文档与代码是否一致；  
- 是否实现了对应难度的主要功能；  
- 是否有值得分享的“AI 使用技巧”或“踩坑经历”。  

鼓励：

- 邀请至少 1 位基础版、1–2 位挑战版同学展示，兼顾不同层次。
